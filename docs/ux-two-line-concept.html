Analysis: Why Two-Line Rows Work

Breaking down the benefits and tradeoffs

Benefits of 2-Line Rows

  • ‚úì Comments are readable: Most comments fit in snippet, no click needed
  • ‚úì Better triage: Can judge relevance without hover/click
  • ‚úì Still fast: 8-10 items is plenty for rapid scanning
  • ‚úì Reduces clicks: Many items can be marked as reviewed after scanning
  • ‚úì Progressive disclosure still works: Hover for even more context if needed
  • ‚úì Differentiation: Easy to spot comments vs issues vs MRs

Tradeoffs Accepted

  • ‚ö† Less density: 8-10 items instead of 12-15 (still good!)
  • ‚ö† More scrolling: To get through 100+ items in Catch-Up Mode
  • ‚ö† Row height: 52px vs 32px (63% taller)
üéØ MVP Recommendation: Two-Line Rows
Start with two-line rows as your MVP default. Here's why:

1. Better User Decisions: With context visible, users can confidently mark items as reviewed or click through to GitLab. No guessing.

2. Comments Shine: Your insight is spot-on - comments are often short enough to read completely in the snippet. This is huge value without any extra clicks.

3. Still Fast: 8-10 items visible is plenty for rapid scanning. You're not losing that much vs 12-15.

4. Room to Optimize: After launch, if users want more density, you can:
‚Ä¢ Add view toggle (1-line vs 2-line)
‚Ä¢ Make row height user-configurable
‚Ä¢ Collapse older items to 1-line (keep recent items 2-line)

5. Hover Tooltips Still Useful: For items where the snippet isn't enough, hover shows the full preview with matched keywords.

The key insight: It's easier to remove context (if users want more density) than to add it (if users feel blind). Start with more context, tune based on real usage.
Implementation Notes
Snippet Generation:
  • Issues/MRs: First 80-100 characters of description (strip markdown formatting)
  • Comments: Full comment text up to ~150 characters (style as italic/quoted)
  • Truncation: Use ellipsis (...) when content exceeds limit
  • Performance: Snippets generated during API polling, stored with event data

Hover Enhancement: Still include hover tooltips for full preview with matched keywords highlighted. Two-line rows + hover = maximum context flexibility.
