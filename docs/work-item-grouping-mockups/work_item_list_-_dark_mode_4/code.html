Implement rate limiting for public API endpoints

High Priority backend security

We've observed a significant increase in bot traffic on the /v1/public/items endpoint. This is causing performance degradation for legitimate users.

GET /v1/public/items?limit=1000
Status: 504 Gateway Timeout
Latency: 12.5s

Requirements:

  • Implement a sliding window rate limiter using Redis.
  • Limit unauthenticated requests to 60 req/min per IP.
  • Return 429 Too Many Requests with standard retry-after headers.
  • Add metrics to track blocked requests in Datadog.
